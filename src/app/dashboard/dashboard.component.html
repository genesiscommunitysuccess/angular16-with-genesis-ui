Developers generally connect to the Genesis Backend via a WebSocket. You can access the authenticated socket url directly from a foundation import. 

{{"import { SOCKET_URL } from '@genesislcap/foundation-auth/env';"}}

The authenticated socket url is: {{ socketUrl }}.

As part of this repo, we've encapsulated @genesislcap/foundation-auth within a module named auth created via the angular-cli.

From there, we re-exported a couple of tokens our application likely needs, while also adding them to the Angular DI container. This was done purely to provide an more comprehensive example, and is entirely optional.

Import and use the value directly from the auth module: 

{{"import { socketUrl } from '../auth/auth.data';"}}

Or import the type from the auth module and engage the Angular DI.

{{"import { SocketUrl } from '../auth/auth.data';"}}
...
@Inject(SocketUrl) public socketUrl: string

The user is also available, and always exist in either an anonymous or authenticated mode.
